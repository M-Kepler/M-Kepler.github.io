# 分治算法

## 算法思想

- `分治法思想`

  求一个规模为 n 的问题，先求左边规模大约为 n/2 的问题，再求右边规模大约为 n/2 的问题，然后合并左边，右边的解，从而求得最终解。

- `分治算法的基本思想`

  是将一个规模为 N 的问题分解为 K 个规模较小的子问题，这些`子问题相互独立且与原问题性质相同`。求出子问题的解，就可得到原问题的解。即一种分目标完成程序算法，简单问题可用二分法完成

## 问题特点

> 将大区间分割成相同的小区间，所有小区间都用相同的方法去求解

- 计算每个小堆钱的方式和计算最大堆钱的==计算方式相同，区别在于体量上==，然后大堆钱总和其实就是小堆钱结果之和

- 把问题分成两个大致相等的子问题，然后递归地对它们求解，这是 **`分`** 的阶段

- **`治`** 阶段就是将两个子问题的解修补到一起并可能再做些少量的附加工作

- 原问题规模比较大，不容易一次求解，一个问题的解可以分解为几个子问题的解

- 问题可以分解为若干规模较小、`求解方式相同，而且相互之间互不影响`的子问题

- 合并子问题就可以得到原问题的解

- 存在递归终止条件，即必须有一个明确的递归结束条件，称之为递归出口

## 解题步骤

对于一个规模为 n 的问题，若该问题可以容易解决则直接求解(`递归退出条件`)，否则将其分解为 k 个规模较小的子问题；这些子问题相互独立且与原问题形式相同，**`递归`** 求解这些子问题，最后所有子问题的解的合并就是原问题的解

- `分`

  将原问题分解成若干个规模较小，`相互独立`，与原问题`形式相同`的子问题

- `治`

  若子问题的规模较小而`容易被解决则直接求解`

- `合并`

  所有子问题的解决结合起来就是原问题的解

## 分治和递归的关系

分治是一种算法策略，递归是这种思想的一种体现（二分查找也可以写成递归和迭代两种形式）
